A. Mengapa menggunakan CSS dan JavaScript dalam pemrograman web modern?

Penggunaan CSS dan JavaScript sangat penting karena mereka membagi tugas pengembangan web menjadi tiga lapisan yang jelas:

  • HTML (Struktur): Bertugas sebagai "rangka" atau fondasi dari halaman web. HTML menentukan elemen apa saja yang ada di halaman, seperti judul, paragraf, gambar, dan tombol.
  • CSS (Presentasi/Tampilan): Bertugas sebagai "kulit" atau desainer. CSS mengatur bagaimana tampilan visual elemen HTML tersebut . Ini mencakup warna , tata letak (layout) , ukuran teks , dan animasi . Tanpa CSS, web hanya akan berupa teks hitam-putih yang berantakan.
  • JavaScript (Interaktivitas): Bertugas sebagai "otak" atau penggerak . JavaScript menambahkan fungsionalitas dan interaksi ke halaman web. Contohnya adalah validasi form, memunculkan popup , mengganti konten secara dinamis (seperti mengganti mode tampilan ), atau mengambil data dari server tanpa me-reload halaman.

Di web modern, pengguna mengharapkan tampilan yang menarik (CSS) dan pengalaman yang responsif/interaktif (JavaScript). Keduanya memisahkan logika dari struktur (HTML), membuat kode lebih bersih, mudah dikelola, dan efisien.

B. Jelaskan konsep dan tujuan utama dari Responsive Web Design (RWD)!

Responsive Web Design (RWD) adalah sebuah pendekatan desain web yang bertujuan agar tampilan situs web dapat beradaptasi secara otomatis (responsif) terhadap berbagai ukuran layar perangkat yang digunakan pengguna .

Konsep utamanya adalah menggunakan "fluid grids" (layout yang fleksibel), "flexible images" (gambar yang ukurannya bisa menyesuaikan), dan "media queries" .

Tujuan utamanya adalah untuk memberikan user experience (UX) yang optimal di semua perangkat—mulai dari smartphone , tablet , hingga desktop —hanya dengan menggunakan satu basis kode HTML dan CSS. Pengguna tidak perlu melakukan zoom-in/zoom-out atau geser kanan-kiri untuk membaca konten, sehingga web lebih nyaman diakses oleh siapa saja, di mana saja.

C. Bagaimana kombinasi CSS, JavaScript, dan prinsip RWD dapat menghasilkan tampilan web yang interaktif sekaligus adaptif?

Ketiganya saling melengkapi untuk menciptakan pengalaman web yang utuh :

  1. RWD (via CSS Media Query) sebagai Fondasi Adaptif: RWD (menggunakan CSS @media ) bertugas mengatur "cetakan" layout berdasarkan ukuran layar. Misalnya, RWD menentukan: "Jika layar desktop , tampilkan 2 kolom. Jika layar smartphone , ubah jadi 1 kolom."
  2. CSS sebagai Penghias (Styling): CSS kemudian mengisi "cetakan" RWD itu dengan gaya . CSS mengatur warna, font , padding, margin , dan memberikan efek transisi agar perpindahan layout RWD terlihat mulus (tidak patah-patah).
  3. JavaScript sebagai Penggerak (Interaktivitas): JavaScript menambahkan interaksi di atas layout adaptif tersebut.

Contoh Sinergi (sesuai tugas ini):

Saat pengguna membuka web di HP (layar kecil ), RWD langsung mengubah layout menjadi satu kolom. CSS memastikan ukuran font dan tombol menjadi lebih besar agar mudah dibaca dan disentuh . Kemudian, pengguna mengklik tombol "Ganti Mode" . JavaScript menangkap klik itu dan mengganti class di <body>. CSS lalu menerapkan aturan gaya yang berbeda (misal: dari 'mode-rapi' ke 'mode-berantakan' ) secara instan. Hasilnya: Tampilan yang sudah adaptif (RWD) tetap bisa interaktif (JavaScript) dan tetap cantik (CSS).
